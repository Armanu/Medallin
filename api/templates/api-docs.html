Medallin Prediction API Edit

This documentation will walk you through the API for Medallin’s Prediction Engine.

Authentication Edit

You need to be authenticated for all API requests. Your requests need to be bearing a valid token for using our services. Contact us at gorjiarman@gmail.com to acquire an API token.

 curl ENDPOINT -H "Authorization: Bearer YOUR_API_TOKEN"

 requests.get(ENDPOINT, data, headers={"Authorization": "Bearer YOUR_API_TOKEN"})

 Axios.get(ENDPOINT, data, { headers: { Authorization: 'Bearer YOUR_API_TOKEN' } } )

Request Statuses Edit

Code Name Description
200 OK Successful.
400 Bad Request Invalid request parameters / missing API token.
401 Unauthorized Invalid or expired API token.
404 Not Found Unknown concept ID (for a disease / symptom).

All successful requests will receive a JSON object as response.

Internationalization Edit

Medallin is a multilingual engine and will process the Accept-Language header of your requests to deduce your preferred display language. You can manually override this property to get the texts and descriptions in your target language. Medallin currently supports English and Persian. Make sure to contact us if you want to help and expand Medallin!

/list/diseases Edit

Lists all the diseases available in the system.

{
  "C0009443": "Common Cold",
  "C0011847": "Diabetes",
  "C0004096": "Asthma",
  ...              
}

/info/disease/ID/ Edit

Returns Medallin's information on the disease identified by "ID".

{
  "label": "Influenza",
  "description": "...",
  "red_flag": false,
  "triage": "low"
}

/predict Edit

Makes a prediction and suggests other symptoms for furthering the examination.

Parameters
symptoms
Object mapping symptom IDs to a value in the range [-1, 1].
profile
Object containing information about the patient.

Making a call to this endpoint will not result in an immediate response. Instead, you will receive an id which you can use to retrieve the results using the endpoint below.

{
  "symptoms": {
      "C0015967": 1,
      "C0010200": -1,
      ...     
  },
  "profile": {
      "age": 26,
      "gender": "male",
      ...        
  }
}

{
  "id": "prediction_id"
}

/predict Edit

Retrieves the result of a prediction request.

Parameters
id
Valid prediction ID.

For more information on the parameter profile, check out the example request below. Lists found in the result are ordered.

{
  "status": "pending",
  "result": null
}

{
  "status": "completed",
  "result": {
    "predictions": [...]
    "suggestions": [...]
  }
}
